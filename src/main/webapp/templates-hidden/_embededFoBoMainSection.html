About This Demo and the FoBo module

This demo demonstrates a optional Lift, generic front-end toolkit module FoBo that quick and easy will integrate some great front-end toolkit(s) into your Lift application, giving you the flexibility and choice of setting up and use the toolkit:s (and version:s) of your liking, that is, if you like you can even setup and potentially use several toolkits simultaneously. For setup information see below.

For more information about the FoBo module take a look at the FoBoAPI and the readme on Github.

Also check out the integrated toolkit documentation for a comprehensive list of the, in this demo included, toolkits features.

Mobile Scalability with FoBo

The FoBo mix - One of the convenience options the FoBo Lift module comes with is the FoBo.InitParam.ToolKit=FoBo.FoBoXX setup option. The FoBoXX setup option is a love, peace and harmony mix of the ZURB/Foundation and the Twitter Bootstrap front-end toolkit. This concert mix will give you the awesomeness of foundation with its rock & roll solid reactive, mobile scalability and the brilliance of bootstrap, it's components and interactions.

This page uses (among others) the FoBo.InitParam.ToolKit=FoBo.FoBo02 option of the FoBo module which make use of the fobo.css, there is also separate BootstrapXX, FoundationXX, JQueryMobileXXX options and more. This demo also make heavy use of a application specific css styling using the fobo02-demo-app1.css file (external to the module).

The ZURB/Foundation toolkit

The ZURB/Foundation toolkit is a great reactive and mobile scalable toolkit.

The FoBo module also gives you the (separate) setup option FoBo.InitParam.ToolKit=FoBo.FoundationXXX for using the foundation toolkit. For a demonstration take a look at the included Foundation demo .

Foundation

The Twitter Bootstrap toolkit

The Twitter Bootstrap toolkit is a great toolkit with lots of candy. It's simple and flexible and have popular user interface components and interactions.

Apart from giving you the easy access and integration of the FoBoXX mix the FoBo module also gives you the (separate) setup option FoBo.InitParam.ToolKit=FoBo.BootstrapXXX to use the bootstrap toolkit. The included bootstrap documentation pages is modified to use the FoBoXX option Bootstrap demo but can as easily (or even more so) be set up and used with the BootstrapXXX option.

Bootstrap

The JQuery Mobile toolkit

The JQuery Mobile toolkit is a Touch-Optimized Web Framework for Smartphones & Tablets.

To use this mobile centric toolkit set FoBo.InitParam.ToolKit=FoBo.JQueryMobileXXX and go mobile.

To demonstrate usage of this mobile centric toolkit the (in this demo) included JQuery Mobile documentation pages is modified to use the Fobo modules FoBo.InitParam.ToolKit=FoBo.JQueryMobile101 option JQuery-mobile demo .

Setup and Usage

For now you have to clone and build the module yourself but thats easy and straight forward so, if you have not done it already, go ahead and do it now, you will find all information you need to prepare the module for usage on Github FoBo module, then come back here for integration and usage.

To initiate and release the powers of the FoBo module chose the Toolkit option(s) you prefer and then call FoBo.init() from your lift projects Boot.

This demo uses the following module setup in lift Boot:


                           import net.liftmodules.FoBo

                           /**
                            * A class that's instantiated early and run.  It allows the application
                            * to modify lift's environment
                            **/
                            class Boot extends Loggable {
                              def boot {

                                //If using defaults FoBo init params can be omitted
                                FoBo.InitParam.JQuery=FoBo.JQuery171  
                                FoBo.InitParam.ToolKit=FoBo.FoBo020 
                                FoBo.InitParam.ToolKit=FoBo.PrettifyJun2011
                                FoBo.InitParam.ToolKit=FoBo.JQueryMobile101
                                FoBo.init()
                                  :

In Lift's html5 template:s add (depending on your setup) something like this:

                        <!DOCTYPE html>
                        <html>
                          <head>
                             :
                             <!--Main css for the FoBo020 module option-->
                             <link href="/classpath/fobo/fobo.css" type="text/css" rel="stylesheet">
                             <!--app specific tweeks and overrides external to module goes here-->
                             <link href="[path to your projects app css]" type="text/css" rel="stylesheet">  
                             :                        
                          </head>
                          <!--Demo page setup for prettyprint and bootstrap scrollspy-->
                          <body data-spy="scroll" data-target="#fobo-navbar" data-offset="40" onload="prettyPrint()">
                             :
                             <!--In end of page (for fast pageload) add the scripts you need--> 
                             <script src="/classpath/fobo/prettify.js" type="text/javascript"></script>
                             <script src="/classpath/fobo/lang-scala.js" type="text/javascript"></script>
                             <script src="/classpath/fobo/bootstrap-scrollspy.js" type="text/javascript"></script>
                             :
                          </body>        
                        </html>

Note the uniform setup without any version references in the html templates, making way for a smooth(er) toolkit upgrade. For more information see the FoBo modules Github.

The FoBo module comes with some convinience functions (snippets, optional to use) that will generate some commonly used toolkit helper scripts for you. For instansce instead of doing this (a common bootstrap component init use case) 

                        <!DOCTYPE html>
                        <html>
                          <head>
                             :
		                  <script type="text/javascript">                                 		
			                function addLoadEvent(func) {
				              var oldonload = window.onload;
				              if (typeof window.onload != 'function') {
					            window.onload = func;
				              } else {
					            window.onload = function() {
						          if (oldonload) {
							        oldonload();
						          }
						          func();
					            }
				              }
			                }
			                addLoadEvent(function() {
				              $('.dropdown-toggle').dropdown()
			                });	   			               
		                  </script>                                 
                          </head>
                          <body">
                             :
                              <!-- Set a boostrap popover element -->   
	                      <a href="#" rel="popover" title="The Title" data-content="The content ..." id="theId">...</a>
                              <!-- Initiate the popover and place it on the left -->
	                      <script>
                                $(function () { 
                                  $("#theId").popover({placement:'left'});
                                }); 
                              </script> 
                              <!-- Prevent the default action for click on the popover -->
                              <script> 
                                $(function(){
                                  $("a[rel=popover]")
                                    .popover()
                                    .click(function(e) {
                                      e.preventDefault()
                                  });
                                });
                              </script> 
                               : 
                          </body>        
                        </html>

You can simply do this using a FoBo:s snippet containing some bootstrap convinience functions: 

                        <!DOCTYPE html>
                        <html>
                          <head>
                               :
                          <script data-lift="FoBo.ScriptHelper.registerLoadEventFactory"></script>         
                          <script data-lift="FoBo.Bootstrap.activateDropdown?on=.dropdown-toggle"></script>              
                          </head>
                          <body">
                               :
	                      <a href="#" rel="popover" title="The Title" data-content="The content ..." id="theId">...</a>
                              <script data-lift="FoBo.Bootstrap.popover?id=#theId&options=placement:'left'"></script>
                              <script data-lift="FoBo.Bootstrap.popoverPreventDefault"></script>
                               : 
                          </body>        
                        </html>

At this writing the available snippet helpers are few but I expect the numbers to grow and you are wellcome to contribute code and/or ideas. Take a look att the API documentation on how to invoke the helpers and description on what result it produces for example look att the FoBo.ScriptHelper.registerLoadEventFactory snippet function by open FoBo.ScriptHelper page and click on the registerLoadEventFactory function name for details.
